# UI 设计稿（结构描述版）

设计要求：这是一个跨平台备忘录应用，中文界面，风格清新简洁，不要复杂花哨。你可以根据现代移动端应用的设计惯例自由发挥配色、圆角、阴影等视觉细节。

---

## 一、首页

### 整体结构

从上到下依次是：
1. **顶部导航栏**：左侧是应用名称"记途"，右侧是设置入口图标。
2. **主体内容区**：三个模块卡片垂直排列，从上到下分别是：待办事项卡片、打卡卡片、笔记卡片。
3. **底部悬浮按钮**：右下角有一个圆形加号按钮，用于快速添加内容。

每个卡片都是可点击的整体区域，点击进入对应模块页面。

### 待办事项卡片

这张卡片代表待办模块。卡片内从上到下包含：
- 一个标题区，有图标和"待办事项"标题
- 一段分隔线
- 四行统计数据，分别显示：总条数、已完成数量、今日截止数量、未完成数量。用数字突出显示关键数据。

### 打卡卡片

这张卡片代表习惯打卡模块。卡片内从上到下包含：
- 标题区，有图标和"习惯打卡"标题
- 分隔线
- 两行统计：今日已打卡几项、今日未打卡几项
- 下方再列出3到4条打卡项的快速预览。每条预览包含：图标、打卡名称、以及右侧的状态（已打卡显示对勾和时长，未打卡显示空白方框）。

### 笔记卡片

这张卡片代表笔记模块。卡片内从上到下包含：
- 标题区，有图标和"笔记"标题
- 分隔线
- 显示"上一次编辑"的说明文字，以及最近一条笔记的标题预览（如果没有笔记则显示空状态提示）

## 二、打卡列表页

### 整体结构

从上到下：
1. **顶部导航栏**：左侧返回按钮，中间标题"习惯打卡"，右侧更多选项按钮。
2. **打卡项列表**：垂直排列的列表，每条打卡项占一行。

列表右下角有一个悬浮加号按钮，点击弹出创建打卡的弹窗。

### 单条打卡项

每条打卡项是一个横向卡片，内部左右分区：

**左侧内容区**（占主要宽度）：
- 第一行：一个图标 + 打卡名称（如"每日读书"）
- 第二行：打卡类型说明。倒计时类型显示"倒计时"和具体时长（如30分钟）；正向计时类型显示"正向计时"；不计时类型显示"不计时"。
- 第三行：累计信息。倒计时和正向计时显示累计时长；不计时显示今日是否已完成。

**右侧操作区**：
- 倒计时和正向计时类型：一个"开始"按钮，点击跳转计时页面。
- 不计时类型：一个"完成"按钮，点击直接完成。
- 如果今日已完成：按钮变为"已完成"，不可点击。

### 打卡计时页（正向计时）

全屏页面。从上到下：
1. 顶部导航栏：返回按钮、打卡名称、更多选项。
2. 标签信息：显示标签名和"正向计时"类型说明。
3. **核心计时显示区**：页面视觉中心，超大字号显示当前计时时间（如00:23:45），这是页面最重要的元素。
4. 统计区：计时下方显示两列数据——今日累计时长、历史累计时长。
5. 主操作按钮："结束打卡"。
6. 下方可列出今日的历史打卡记录。

### 打卡计时页（倒计时）

全屏页面。结构类似正向计时，但有以下差异：
1. 标签显示"倒计时"。
2. 核心显示区显示剩余时间（如00:12:30）。
3. 时间下方有一条进度条，显示剩余进度。
4. 进度条下方显示计划总时长。
5. 主操作按钮"结束打卡"下方，有三个快捷加时按钮：+5分、+10分、+30分。

**倒计时结束状态**：时间显示为0，进度条满格，显示"任务完成"提示。
**超时状态**：显示已超时的时长（如"已超时5分30秒"），进度条变色警示，快捷加时按钮突出显示。

### 创建打卡弹窗

屏幕中央弹出的卡片式弹窗。从上到下：
1. 弹窗头部：左侧"新建打卡"标题，右侧关闭按钮。
2. 输入框：输入打卡名称，提示文字"输入打卡内容"。
3. 标签选择行：文字"标签"，后面是一个下拉选择框，可以选择标签。
4. 类型选择：三个选项排成一排，分别是"倒计时"、"正向计时"、"不计时"，单选。
5. 条件显示区：
   - 如果选"倒计时"：出现时长选择，有"10分钟"、"30分钟"、"自定义"三个快捷选项。选"自定义"时出现一个数字输入框输入分钟数。
   - 如果选"正向计时"：显示一句提示"适合碎片时间记录"。
   - 如果选"不计时"：显示一句提示"适合不需要计时的打卡内容，比如吃药"。
6. 底部按钮：左侧"取消"，右侧"创建"。

## 三、笔记列表页

### 整体结构

从上到下：
1. **顶部导航栏**：返回按钮、"笔记列表"标题、设置按钮。
2. **搜索栏**：一个搜索输入框，带搜索图标，右侧有一个排序筛选按钮。
3. **笔记瀑布流**：卡片以瀑布流形式排列（类似Pinterest）。移动端两列，桌面端三到四列。每张笔记卡片高度根据内容自适应。

右下角有悬浮加号按钮，点击进入新建笔记页面。

### 单条笔记卡片

每张卡片是一个垂直结构：
- **图片区（可选）**：如果笔记里有图片，卡片顶部显示一张图片缩略图。没有图片则直接显示文字内容。
- **标题区**：笔记标题，单行显示，超出省略。如果用户没写标题，自动取正文前几个字。
- **内容摘要**：正文的纯文字摘要，两到四行，超出截断。
- **时间区**：底部显示最后编辑时间。

置顶的笔记卡片需要有明显的置顶视觉标记（如一个小图标或特殊边框）。

### 笔记编辑页

全屏页面。从上到下：
1. **顶部标题栏**：左侧返回按钮，中间是标题输入框（提示"标题（可选）"），右侧完成按钮。
2. **富文本编辑区**：占页面主要区域的大块空白编辑区域，提示文字"开始记录..."。支持输入文字、换行、插入图片。
3. **底部工具栏**：固定在底部，横向排列格式按钮：加粗、斜体、下划线、插入图片、插入附件。

## 四、待办列表页

### 整体结构

从上到下：
1. **顶部导航栏**：返回按钮、"待办事项"标题、更多选项。
2. **搜索筛选区**：左侧是一个搜索框（带搜索图标），右侧是一个类型筛选下拉框（如"全部类型"）。
3. **待办项列表**：垂直排列，每条待办占一行。

右下角有悬浮加号按钮，点击弹出创建待办弹窗。

### 单条待办项

每条待办项横向排列，从左到右：
1. **复选框**：左侧，可点击切换完成状态。未完成是空方框，已完成是选中状态。
2. **内容区**：复选框右侧，垂直排列：
   - 第一行：待办标题，单行。已完成时文字变灰并带删除线。
   - 第二行：截止日期和优先级。优先级分高、中、低三级，用颜色区分。
3. **展开备注**：点击条目可展开，下方显示备注内容。

置顶项有明显的置顶标记。

### 创建待办弹窗

屏幕中央弹出的卡片式弹窗。从上到下：
1. 弹窗头部："新建待办"标题和关闭按钮。
2. **输入行**：横向排列三个元素——左侧装饰性复选框（不可点）、中间输入框（提示"回车可连续添加待办事项"）、右侧类型下拉框（如"工作"）。
3. **待办清单区**：当用户输入内容按回车后，输入框上方展开一个清单列表，显示已添加的条目。每个条目前面有复选框。清单区可左滑删除条目。
4. **优先级选择**："高"、"中"、"低"三个单选选项。
5. **底部按钮栏**：左侧是"设置日期"按钮（带闹钟图标），点击弹出日期选择器。右侧是"完成"按钮，点击批量创建所有清单条目并关闭弹窗。

## 五、设置页

### 5.1 整体结构

从上到下：
1. **顶部导航栏**：左侧返回按钮，中间标题"设置"，右侧为空。
2. **设置列表**：垂直排列的分组列表，同类功能归为一组，组间有分组标题。每组内是若干列表项。

### 5.2 列表项通用结构

每条列表项横向排列：
- 左侧：一个图标 + 功能名称（如"导出数据"）。
- 右侧：状态文字（灰色小字，如"未登录"、"自动同步: 开启"）+ 右箭头，表示可点击进入子页面。
- 开关类设置（如自动同步）右侧用开关控件代替箭头和状态文字。

### 5.3 分组一：账户与云端

这个分组在未登录时只显示"登录账号"一项，右侧状态"未登录"。

登录后显示三项：
- **云端同步**：右侧状态显示"自动同步: 开启/关闭"，点击进入云端同步详情页。
- **云端数据管理**：右侧状态显示"最近备份: 昨天"，点击进入备份记录列表页。
- **退出登录**：红色文字，无箭头，点击弹出确认弹窗。

### 5.4 分组二：数据管理

包含三项：
- **导出数据**：点击进入导出选项弹窗，选择格式（CSV/TXT/JSON）和导出内容（笔记/待办/打卡/全部）。
- **导入数据**：点击弹出文件选择器，选择文件后显示导入预览（条数、冲突提示）和导入策略选择。
- **回收站**：右侧状态显示回收站内待恢复条数（如"3条待恢复"），点击进入回收站页面。

### 5.5 分组三：个性化

包含三项，均点击进入子页面或弹出选项：
- **主题**：右侧状态显示当前主题（如"跟随系统"），选项为浅色/深色/跟随系统。
- **字体设置**：右侧状态显示当前字号（如"标准"），可调整字体大小和字体样式。
- **提醒音效**：右侧状态显示"已开启"或"已关闭"，可分别设置待办提醒音效、打卡倒计时音效、震动开关。

### 5.6 分组四：安全与隐私

包含一项：
- **应用锁**：右侧状态显示"未开启"，点击进入设置启动密码（数字密码或图案密码）。开启后状态变为"已开启"。

### 5.7 分组五：其他

包含三项：
- **使用帮助**：点击进入帮助页面，包含新手指引、常见问题、操作教程。
- **分享应用**：点击调用系统分享面板。
- **关于**：右侧显示版本号（如"v1.0.0"），点击进入关于页面，显示应用Logo、版本、检查更新、隐私政策。

### 5.8 云端同步详情页

全屏页面。从上到下：
1. 顶部导航栏：返回按钮 + "云端同步"标题。
2. **自动同步开关**：一个大号开关控件，下方文字说明。
3. **同步内容选择**：多选列表，可选笔记、待办、打卡、设置偏好。
4. **同步状态信息**：显示上次同步时间和同步状态。
5. **立即同步按钮**：大号主操作按钮，手动触发同步。
6. **仅在Wi-Fi下同步开关**：单独的开关选项。

### 5.9 云端数据管理页

全屏页面。从上到下：
1. 顶部导航栏：返回按钮 + "云端数据管理"标题。
2. **存储空间概览**：一个进度条显示已用空间占比，下方文字说明。
3. **备份记录列表**：按时间倒序排列。每条记录显示备份时间、条数、类型（自动/手动），右侧有"下载"和"删除"两个操作按钮。
4. **底部操作区**：固定在底部，两个按钮并列——"全量下载至本地"和"清理旧备份"。

### 5.10 回收站页面

全屏页面。从上到下：
1. 顶部导航栏：返回按钮 + "回收站"标题 + 清空按钮。
2. **模块标签栏**：横向排列的标签页，可选"全部"、"笔记"、"待办"、"打卡"。
3. **删除项列表**：每条显示原条目名称和删除时间，左侧有复选框可多选。
4. **底部操作栏**：选中条目时底部出现操作栏，两个按钮——"恢复"和"彻底删除"。

## 六、通用设计建议（供AI参考）

- 整体视觉语言保持统一，三个模块可以用不同的主题色来区分，但不要太花哨。
- 打卡模块建议用绿色系（与习惯养成的积极感一致），待办用蓝色系（清爽高效），笔记用暖色系（温暖亲和）。
- 卡片式设计，有适当的留白，不要挤在一起。
- 重要操作按钮（如"创建"、"完成"、"结束打卡"）需要突出显示。
- 已完成的项（如已打卡、已完成待办）视觉上要弱化（如变灰、降低对比度），与未完成项形成区分。
- 计时页面中，时间数字是绝对的视觉焦点，应该最大最醒目。
- 设置页列表项的图标风格保持一致，用简洁的线性图标或面性图标。
- 危险操作（如退出登录、彻底删除、清空回收站）用红色文字或按钮，与常规操作形成区分。
- 开关控件用平台原生样式或统一自定义样式，开启状态用主色调。
