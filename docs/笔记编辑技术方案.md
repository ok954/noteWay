# 笔记编辑技术方案

## 一、技术选型

| 层级 | 选型 | 说明 |
|------|------|------|
| 富文本编辑器 | `flutter_quill` | Flutter 富文本编辑事实标准，内置支持加粗/斜体/下划线/图片插入，跨端体验一致 |
| 本地数据库 | `sqflite` | 结构化存储，支持高效查询、排序、搜索，适合笔记列表场景 |
| 图片存储 | 应用私有目录 + 数据库存路径 | 避免数据库膨胀，便于独立管理图片生命周期 |
| 状态管理 | `Riverpod` / `Provider` | 编辑状态与首页列表实时同步 |

---

## 二、数据结构设计

### 2.1 笔记表（notes）

```sql
CREATE TABLE notes (
    id TEXT PRIMARY KEY,           -- UUID
    title TEXT,                    -- 标题，用户未填写时取正文前10字
    content TEXT NOT NULL,         -- Quill Delta JSON 字符串
    plain_text TEXT,               -- 纯文本摘要，用于搜索预览和标题自动生成
    image_paths TEXT,              -- 关联图片路径列表，JSON 数组序列化存储
    created_at INTEGER NOT NULL,   -- 创建时间戳（毫秒）
    updated_at INTEGER NOT NULL    -- 更新时间戳（毫秒）
);
```

### 2.2 图片存储路径

```
<应用私有目录>/
  ├─ images/
  │   ├─ note_<uuid>_img_<timestamp>.jpg
  │   └─ ...
  └─ database.db
```

- 图片插入时复制到应用私有目录并重命名，防止原文件被外部删除导致引用失效
- 数据库仅存储相对路径，不存二进制数据
- 删除笔记时同步清理对应图片文件

## 三、实时保存机制

采用**防抖保存（Debounce）**策略，平衡实时性与 IO 性能。

```dart
Timer? _saveTimer;

void onContentChanged() {
  _saveTimer?.cancel();
  _saveTimer = Timer(const Duration(seconds: 1), () {
    _persistNote();
  });
}

// 退出编辑页时强制保存
void dispose() {
  _saveTimer?.cancel();
  _persistNote();
}
```

| 场景 | 行为 |
|------|------|
| 用户持续输入 | 等待停止输入 1 秒后触发保存 |
| 用户快速退出页面 | `dispose()` 中取消定时器并立即强制保存 |
| 应用异常崩溃 | 下次启动时检查草稿缓存，提示恢复 |

## 四、空闲后自动上传云端

本地保存成功后，若用户一段时间内未再次编辑，自动将笔记上传至云端备份。

### 4.1 双定时器机制

```dart
Timer? _localSaveTimer;   // 本地保存防抖（1秒）
Timer? _cloudUploadTimer; // 云端上传空闲检测（30秒）

void onContentChanged() {
  // 1. 取消云端上传定时器，用户仍在编辑
  _cloudUploadTimer?.cancel();

  // 2. 本地防抖保存
  _localSaveTimer?.cancel();
  _localSaveTimer = Timer(const Duration(seconds: 1), () async {
    await _persistNote();        // 写入本地数据库
    _startCloudUploadTimer();    // 本地保存成功后启动云端空闲检测
  });
}

void _startCloudUploadTimer() {
  _cloudUploadTimer?.cancel();
  _cloudUploadTimer = Timer(const Duration(seconds: 30), () async {
    await _tryUploadToCloud();   // 30秒内无编辑，尝试上传
  });
}
```

### 4.2 上传触发条件

| 条件 | 说明 |
|------|------|
| **用户已登录** | 未登录状态不上传，保持功能文档"默认无需登录"原则 |
| **自动同步已开启** | 用户在设置中开启云端自动同步开关 |
| **网络可用** | 检测网络状态，无网络时不触发上传 |
| **数据有变更** | 对比本地 `updated_at` 与上次上传时间，无变化则跳过 |

### 4.3 上传流程

```
空闲 30 秒到期
   ↓
检查登录状态 + 自动同步开关 + 网络状态
   ↓
条件不满足 → 记录待上传标记，下次满足时补传
条件满足   → 开始上传
   ↓
将笔记内容（Delta JSON、标题、图片）打包
   ↓
调用云端上传 API
   ↓
上传成功 → 更新本地 "last_synced_at" 时间戳，清除待上传标记
上传失败 → 标记待上传状态，下次空闲或网络恢复时重试
```

### 4.4 图片上传策略

笔记中包含图片时，上传流程如下：

1. **先传图片**：将本地图片文件逐个上传至云存储（如 OSS / S3），获取云端 URL。
2. **再传笔记**：将笔记内容中的本地图片路径替换为云端 URL，再上传笔记 JSON 数据。
3. **本地保留原路径**：云端 URL 仅用于同步，本地数据库仍存储私有目录路径，保证离线时正常显示。
4. **失败处理**：图片上传失败时，整条笔记标记为"部分同步"，下次重试时仅上传失败的部分。

### 4.5 冲突处理（简单时间戳策略）

| 场景 | 处理方式 |
|------|----------|
| 本地 `updated_at` > 云端 `updated_at` | 本地覆盖云端 |
| 本地 `updated_at` < 云端 `updated_at` | 云端覆盖本地（提示用户有更新） |
| 本地 `updated_at` == 云端 `updated_at` | 跳过，无需同步 |

### 4.6 边界处理

| 场景 | 处理策略 |
|------|----------|
| **编辑中切后台** | App 生命周期监听，进入后台时立即强制本地保存 + 触发云端上传（若满足条件） |
| **快速连续编辑** | 每次编辑取消云端定时器，只有真正停止编辑 30 秒后才上传，避免频繁请求 |
| **上传中用户又开始编辑** | 当前上传继续完成，不中断；新的编辑内容将在下次空闲时再次上传 |
| **断网后恢复** | 网络状态监听，恢复时检查待上传队列，批量补传 |
| **大图上传慢** | 上传过程在后台 isolate 执行，不阻塞 UI；上传进度可透传给设置页的云端管理 |

## 五、核心实现流程

### 5.1 新建/打开笔记

```
进入编辑页
   ↓
根据 noteId 查询数据库
   ↓
有数据 → 加载 Quill Delta JSON 到 QuillController
无数据 → 初始化空 QuillController
   ↓
监听文档变化（onContentChanged）
```

### 5.2 保存流程

```
用户停止输入 1 秒
   ↓
从 QuillController 提取 Delta
   ↓
Delta 序列化为 JSON String
   ↓
提取纯文本（用于搜索和标题生成）
   ↓
若标题为空 → 取 plainText.substring(0, 10) 作为默认标题
   ↓
写入 sqflite（insert 或 update）
   ↓
通知首页状态更新（Provider/Riverpod）
```

### 5.3 图片插入流程

```
用户选择图片（相册/相机）
   ↓
复制图片到 <私有目录>/images/，生成唯一文件名
   ↓
通过 Quill Embed 插入图片块（图片路径为私有目录路径）
   ↓
触发防抖保存，image_paths 字段更新
```

---

## 五、关键边界处理

| 场景 | 处理策略 |
|------|----------|
| **空笔记丢弃** | 标题和内容均为空时，退出编辑页不保存，避免产生无意义记录 |
| **标题自动生成** | 实时监听纯文本变化，用户未手动输入标题时，自动截取正文前 10 字 |
| **图片清理** | 删除笔记时遍历 `image_paths`，删除对应物理文件；避免数据库与文件系统不一致 |
| **草稿恢复** | 编辑过程中崩溃，将未保存的 Delta 临时写入 `SharedPreferences`，下次进入时检测并提示恢复 |
| **大数据量** | `plain_text` 字段用于搜索和列表预览，避免加载完整 Delta 到内存；列表页仅查询 id/title/updated_at |
