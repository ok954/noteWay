# 设置页需求与技术方案

## 一、页面定位

设置页为应用全局配置入口，集中管理**数据安全**、**云端同步**、**个性化偏好**、**应用信息**四大类功能。用户从首页右上角设置图标或各页面导航进入。

---

### 2.2 分组说明

| 分组 | 说明 |
|------|------|
| **账户与云端** | 登录状态、云端同步开关、云端数据管理。未登录时仅显示登录入口，其余隐藏。 |
| **数据管理** | 数据导出、导入、回收站。核心数据安全操作集中在此。 |
| **个性化** | 主题、字体、音效。纯本地偏好设置。 |
| **安全与隐私** | 应用锁。本地数据加密保护。 |
| **其他** | 使用帮助、分享、关于。 |

### 2.3 列表项结构

每条列表项左侧为图标 + 主标题，右侧为状态文字 + 右箭头 `>`：
- 主标题：黑色中等字，说明功能名称。
- 状态文字（可选）：灰色小字，显示当前设置状态。如"未登录"、"自动同步: 开启"。
- 右箭头：表示点击进入子页面或弹窗。

开关类设置（如自动同步）右侧显示开关控件（Switch），不显示箭头。

## 三、功能详情

### 3.1 登录账号

**入口**：未登录时显示"登录账号"，已登录时显示当前账号信息（如手机号/微信昵称）。

**点击行为**：
- 未登录 → 打开登录弹窗/页面。
- 已登录 → 打开账号管理页面（显示登录方式、退出登录按钮）。

**登录方式**：
- 手机号 + 验证码
- 微信快捷登录
- QQ 快捷登录

**登录后变化**：
- 设置页显示"云端同步"、"云端数据管理"入口。
- 笔记编辑页的空闲上传逻辑生效。
- 首页及设置页头像/账号信息更新。

**功能说明**：

- **仅在Wi-Fi下同步**：开启后，移动网络下不自动同步，但可手动触发。

### 3.3 云端数据管理

**前提**：用户已登录。

**页面内容**：

从上到下：
1. **存储空间概览**：进度条形式显示已用空间/总空间。
2. **备份记录列表**：按时间倒序排列的备份记录。
   - 每条记录显示：备份时间、备份条数、备份类型（自动/手动）。
   - 每条记录右侧有"下载"和"删除"操作。
3. **底部按钮**："全量下载至本地"、"清理旧备份"（保留最近10条）。

**操作逻辑**：
- **下载**：选择备份记录，将云端数据覆盖或合并到本地。
- **删除**：删除云端该条备份记录。
- **全量下载**：下载最新完整备份到本地，与本地数据按时间戳冲突处理。

### 3.4 导出数据

**交互流程**：

```
点击"导出数据"
   ↓
弹出导出选项弹窗
   ↓
选择导出格式（CSV / TXT / JSON）
选择导出内容（笔记 / 待办 / 打卡 / 全部）
   ↓
点击"导出"
   ↓
弹出系统文件选择器，选择保存路径
   ↓
生成文件，提示"导出成功" + 文件路径
```

**导出格式说明**：
- **CSV**：表格形式，适合Excel打开。笔记导出标题+纯文本；待办导出标题+状态+日期；打卡导出名称+累计时长。
- **TXT**：纯文本，适合阅读。按模块分组，人类可读格式。
- **JSON**：完整结构化数据，适合备份或迁移。包含所有字段，后续可原样导入。

**边界处理**：
- 笔记中的图片：JSON格式导出时包含图片路径；CSV/TXT格式仅导出纯文本，图片不跟随导出。
- 大数据量导出：显示进度条，后台生成文件，不阻塞UI。

---

### 3.5 导入数据

**交互流程**：

```
点击"导入数据"
   ↓
弹出系统文件选择器，选择之前导出的文件
   ↓
解析文件格式
   ↓
显示预览：将要导入的条数、冲突项提示
   ↓
选择导入策略：
   - 智能合并（推荐）：无冲突直接导入，有冲突保留最新
   - 全部覆盖：云端/本地数据被导入内容覆盖
   - 仅新增：跳过已有内容，只导入新条目
   ↓
点击"确认导入"
   ↓
执行导入，显示进度，完成后提示成功条数
```

**去重规则**：
- 按数据内容哈希 + 时间戳综合判断是否为重复。
- 笔记：标题+正文前100字+创建时间。
- 待办：标题+截止日期。
- 打卡：打卡名称+类型。

---

### 3.6 回收站

**页面结构**：按模块分标签页（笔记 / 待办 / 打卡）。

**列表项**：显示被删除条目的名称、删除时间。左侧有复选框，可多选。

**操作按钮**：
- **恢复**：将选中项恢复到原位置。
- **彻底删除**：永久删除选中项。
- **清空回收站**：一键删除所有内容。

**自动清理**：删除超过7天的条目自动彻底删除。

---

### 3.7 清除所有数据

**入口**：设置页"数据管理"分组最后一项，文字为红色警示色。

**适用场景**：
- 测试阶段产生大量无效数据，希望一键重置
- 换用账号前清空旧数据
- 应用转让给其他人使用前

**点击行为**：

```
点击"清除所有数据"
   ↓
弹出二次确认弹窗，列出将被删除的内容清单
   ↓
用户点击"确认清除"
   ↓
显示清除进度（约1-2秒）
   ↓
清除完成后自动返回首页，所有 Provider 状态重置
```

**二次确认弹窗内容**：

```
┌────────────────────────────┐
│  ⚠️ 确认清除所有数据       │
├────────────────────────────┤
│                            │
│  此操作将永久删除：        │
│  · 所有笔记及关联图片      │
│  · 所有待办事项            │
│  · 所有打卡项及打卡记录    │
│  · 回收站内容              │
│  · 所有设置偏好            │
│                            │
│  ⚠️ 此操作不可恢复！       │
│                            │
│  [取消]    [确认清除]      │
└────────────────────────────┘
```

**清除范围**：

| 数据类型 | 处理方式 | 说明 |
|----------|----------|------|
| 笔记内容 | 删除 notes 表全部数据 | 文字内容 |
| 笔记图片 | 删除应用私有目录 images/ 下所有文件 | 物理文件 |
| 待办事项 | 删除 todos 表全部数据 | |
| 待办类型 | 重置为内置3个类型 | 清除自定义类型 |
| 打卡项 | 删除 habits 表全部数据 | |
| 打卡记录 | 删除 checkin_records 表全部数据 | |
| 标签 | 重置为内置3个标签 | |
| 回收站 | 删除 recycle_bin 表全部数据 | |
| 设置偏好 | 清空 SharedPreferences | 主题、字体等恢复默认 |
| 本地通知 | 取消所有已注册通知 | |

**技术实现**：详见 [数据清除与卸载方案](数据清除与卸载方案.md) 中的 `DataClearService` 实现。

**重置后状态**：
- 数据库表清空并重新插入初始化数据（内置类型、标签）
- 所有 Riverpod Provider 状态重置
- 页面自动跳转回首页
- 应用恢复为首次安装时的初始状态

## 五、数据表设计

### 5.1 设置配置表（settings）

```sql
CREATE TABLE settings (
    key TEXT PRIMARY KEY,          -- 配置项键名
    value TEXT,                    -- 配置项值
    updated_at INTEGER             -- 更新时间
);

-- 常用配置项
-- theme: light/dark/system
-- font_size: small/standard/large/extra_large
-- font_family: system/songti/heiti
-- auto_sync: 0/1
-- wifi_only_sync: 0/1
-- todo_remind_sound: 0/1
-- habit_remind_sound: 0/1
-- vibration: 0/1
-- app_lock_enabled: 0/1
-- app_lock_type: pin/pattern
```

### 5.2 回收站表（recycle_bin）

```sql
CREATE TABLE recycle_bin (
    id TEXT PRIMARY KEY,
    item_type TEXT NOT NULL,       -- note / todo / habit
    item_id TEXT NOT NULL,         -- 原条目ID
    item_data TEXT NOT NULL,       -- 原条目JSON数据
    deleted_at INTEGER NOT NULL,   -- 删除时间
    expires_at INTEGER NOT NULL    -- 自动清理时间（deleted_at + 7天）
);
```

### 5.3 云端同步记录表（sync_records）

```sql
CREATE TABLE sync_records (
    id TEXT PRIMARY KEY,
    item_type TEXT NOT NULL,       -- note / todo / habit / setting
    item_id TEXT NOT NULL,         -- 本地条目ID
    sync_action TEXT NOT NULL,     -- upload / download / conflict
    sync_status TEXT NOT NULL,     -- pending / success / failed
    synced_at INTEGER,             -- 同步成功时间
    error_msg TEXT                 -- 失败原因
);
```
