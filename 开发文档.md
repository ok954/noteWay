# 记途 — 开发文档

## 一、前期准备

### 1.1 开发环境准备

#### 1.1.1 Flutter 开发环境

| 工具 | 要求 | 安装说明 |
|------|------|----------|
| Flutter SDK | ≥ 3.16，Dart ≥ 3.0 | 官网下载稳定版，配置 `flutter` 环境变量 |
| Android Studio | 最新版 | 安装 Flutter + Dart 插件，配置 Android SDK |
| Xcode | ≥ 15（Mac 必需） | iOS 开发及真机调试必需 |
| VS Code | 最新版 | 安装 Flutter、Dart、GitLens 插件 |
| Git | ≥ 2.40 | 版本控制 |

**环境验证命令**：
```bash
flutter doctor          # 检查环境是否完整
flutter --version       # 确认版本
```

**常见问题**：
- Windows 开发 iOS：无法直接编译 iOS，需 Mac 或远程构建服务
- Android 模拟器：建议用 Android Studio 自带模拟器，或真机调试
- Windows 桌面端：`flutter config --enable-windows-desktop`

#### 1.1.2 服务端开发环境（如自建后端）

| 工具 | 要求 | 说明 |
|------|------|------|
| Node.js | ≥ 18 LTS | NestJS 运行基础 |
| pnpm / npm | 最新版 | 包管理工具 |
| PostgreSQL | ≥ 15 | 主数据库 |
| Redis | ≥ 7 | 缓存 + 会话 |
| Docker + Docker Compose | 最新版 | 本地一键启动依赖服务 |

**Docker Compose 快速启动（开发环境）**：
```yaml
# docker-compose.dev.yml
version: '3.8'
services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: memo_db
      POSTGRES_USER: dev
      POSTGRES_PASSWORD: dev123
    ports:
      - "5432:5432"
    volumes:
      - pg_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  minio:
    image: minio/minio
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minio
      MINIO_ROOT_PASSWORD: minio123
    ports:
      - "9000:9000"
      - "9001:9001"
    volumes:
      - minio_data:/data

volumes:
  pg_data:
  minio_data:
```

启动命令：
```bash
docker-compose -f docker-compose.dev.yml up -d
```

## 三、开发流程

### 3.1 开发顺序建议

按模块优先级和依赖关系，建议按以下顺序开发：

| 阶段 | 模块 | 优先级 | 说明 |
|------|------|--------|------|
| **Phase 1** | 数据库层 + 基础组件 | P0 | 所有模块的根基，先搭好 |
| **Phase 2** | 首页 + 笔记模块 | P0 | 笔记功能独立，可最早交付 |
| **Phase 3** | 待办模块 | P0 | 与笔记平级，互不依赖 |
| **Phase 4** | 打卡模块 + 计时功能 | P0 | 核心功能，依赖数据库层 |
| **Phase 5** | 设置页 + 导入导出 | P1 | 数据管理功能 |
| **Phase 6** | 云端同步 + 登录 | P1 | 需要服务端配合 |
| **Phase 7** | 回收站 + 应用锁 | P2 | 进阶功能 |
| **Phase 8** | 主题/字体/音效 | P2 | 个性化功能 |

### 3.2 各模块开发要点

#### 3.2.1 数据库层（Phase 1）

**开发内容**：
- [ ] 创建 `DatabaseHelper` 单例，封装 open/close/transaction
- [ ] 实现所有表的创建 SQL
- [ ] 实现 `BaseRepository<T>` 抽象类（增删改查、批量操作）
- [ ] 实现各模块 Repository：
  - `NoteRepository`
  - `TodoRepository`
  - `HabitRepository`
  - `CheckinRecordRepository`
  - `SettingsRepository`

**关键代码模式**：
```dart
abstract class BaseRepository<T> {
  Future<int> insert(T item);
  Future<int> update(T item);
  Future<int> delete(String id);
  Future<T?> getById(String id);
  Future<List<T>> getAll();
}
```

#### 3.2.2 首页（Phase 2）

**开发内容**：
- [ ] 三个统计卡片 UI（待办/打卡/笔记）
- [ ] 各模块数据查询（总条数、已完成、今日状态）
- [ ] 卡片点击跳转对应模块
- [ ] FAB 快速添加入口（弹窗选择添加类型）

**注意**：首页数据刷新用 Riverpod `FutureProvider` 或 `StreamProvider`，自动响应数据变化。

#### 3.2.3 笔记模块（Phase 2）

**开发内容**：
- [ ] 笔记列表页：瀑布流布局（`MasonryGridView`）
- [ ] 笔记卡片组件：图片缩略图、标题、摘要、时间
- [ ] 笔记编辑页：`flutter_quill` 集成
- [ ] 实时保存：1 秒防抖 + 退出强制保存
- [ ] 图片插入：选择 → 复制到私有目录 → Delta Embed
- [ ] 搜索功能：标题 + 正文纯文本模糊匹配

**关键技术点**：
- Quill Delta 转纯文本用于搜索：`document.toPlainText()`
- 图片文件生命周期：插入时复制，删除笔记时清理孤儿文件
- 空笔记丢弃：标题和内容都为空时退出不保存

#### 3.2.4 待办模块（Phase 3）

**开发内容**：
- [ ] 待办列表页：线性列表 + 搜索筛选栏
- [ ] 单条待办项：复选框 + 标题 + 截止日期 + 优先级
- [ ] 完成动画：300ms 斜线置灰 → 400ms 移动到底部
- [ ] 展开/收起备注：点击展开显示备注内容
- [ ] 创建弹窗：连续添加模式（回车添加清单）、日期选择器、优先级选择
- [ ] 类型管理：内置 3 个 + 最多 2 个自定义类型
- [ ] 排序逻辑：置顶 → 今日截止 → 截止日期 → 优先级 → 编辑时间

**关键技术点**：
- 完成动画用 `AnimatedContainer` + `AnimatedOpacity`
- 日期选择用 `showDatePicker` + `showTimePicker`
- 类型编辑弹窗的输入校验：非空、不重复、不超过 4 字

#### 3.2.5 打卡模块（Phase 4）

**开发内容**：
- [ ] 打卡列表页：线性列表，三种类型展示
- [ ] 单条打卡项：图标 + 名称 + 类型标签 + 累计时长 + 操作按钮
- [ ] 打卡计时页（正向计时）：超大时间显示 + 统计 + 结束按钮
- [ ] 打卡计时页（倒计时）：剩余时间 + 进度条 + 快捷加时
- [ ] 创建打卡弹窗：名称 + 标签 + 类型选择 + 条件时长/提示
- [ ] 计时核心逻辑：记录时间戳，进入页面计算，不依赖后台 Timer
- [ ] 本地通知：`flutter_local_notifications` 注册倒计时结束提醒
- [ ] 每日重置：应用启动检测日期变化，重置今日数据

**关键技术点**：
- 计时状态持久化：`habits.is_timing` + `current_record_id`
- 切后台/杀进程恢复：根据 `start_time` / `end_time` 重新计算
- 倒计时结束提醒用 `zonedSchedule` 注册精确时间通知

#### 3.2.6 设置页（Phase 5）

**开发内容**：
- [ ] 分组列表 UI：5 个分组，列表项统一组件
- [ ] 导入导出：文件选择 → 格式解析 → 数据写入/读出
- [ ] 回收站：按模块分标签页，恢复/彻底删除/自动清理
- [ ] 主题切换：`AdaptiveTheme` 或自研 ThemeProvider，切换后立即生效

#### 3.2.7 云端同步（Phase 6）

**开发内容**：
- [ ] 登录功能：手机号验证码 / 微信 / QQ
- [ ] 同步协议实现：增量上传/下载 + 时间戳冲突处理
- [ ] 空闲上传：双定时器（1s 本地防抖 + 30s 云端空闲检测）
- [ ] 网络监听：`connectivity_plus` 检测网络状态变化
- [ ] 服务端 API 对接：登录、上传、下载、备份列表

**服务端接口清单**（如自建后端）：
```
POST /api/auth/login          # 登录（手机号/第三方）
POST /api/auth/refresh        # 刷新 Token
GET  /api/auth/profile        # 获取用户信息

POST /api/sync/upload         # 增量上传
POST /api/sync/download       # 增量下载
GET  /api/sync/backups        # 备份记录列表
POST /api/sync/backups/:id    # 按备份下载
DELETE /api/sync/backups/:id  # 删除备份

POST /api/upload/image        # 图片上传
```

## 六、开发规范

### 6.1 代码规范

- **Dart 代码风格**：遵循 `flutter_lints`，`dart format` 自动格式化
- **命名规范**：
  - 类名：大驼峰 `NoteRepository`
  - 方法/变量：小驼峰 `getNoteById`
  - 常量：全大写下划线 `MAX_TITLE_LENGTH`
  - 文件：小写下划线 `note_repository.dart`
- **注释**：仅写 WHY，不写 WHAT。复杂逻辑加简短注释。

### 6.2 Git 工作流

```
main          ← 生产分支，只能发 PR 合并
  ↓
develop       ← 开发分支，日常合并
  ↓
feature/xxx   ← 功能分支，从 develop 切出
  ↓
PR → Code Review → 合并到 develop
```

### 6.3 提交规范

```
feat: 添加笔记搜索功能
```

## 七、常见问题排查

| 问题 | 排查方向 |
|------|----------|
| sqflite 在 Windows 报错 | 确认使用 `sqflite_common_ffi`，已初始化 FFI |
| flutter_quill 图片不显示 | 检查图片路径是否为绝对路径，文件是否存在 |
| 本地通知不触发 | 检查权限申请、通知渠道配置（Android 8+）、精确闹钟权限 |
| 导入导出文件找不到 | 检查 `path_provider` 获取的路径，`file_picker` 版本兼容性 |
| 云端同步数据丢失 | 检查时间戳时区是否统一（全部用 UTC 或全部用本地） |
| iOS 打包签名失败 | 检查 Apple Developer 账号、Bundle ID、Provisioning Profile |
