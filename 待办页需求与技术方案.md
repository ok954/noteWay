# 待办页需求与技术方案

## 一、页面布局

### 1.1 列表形式
- 采用**线性列表（ListView）**，每个待办项独占一行。
- 每个待办项卡片支持展开/收起，展开后展示备注内容。

### 1.2 悬浮按钮
- 页面右下角固定一个**圆形悬浮加号按钮（FAB）**。
- 点击后打开待办创建弹窗。

---

## 二、待办项展示

### 2.1 单条待办项结构

```
┌─────────────────────────────────────┐
│ [复选框]  待办事项标题              │
│          截止日期  |  优先级标签     │
│  备注内容摘要...                    │
└─────────────────────────────────────┘
```

- **复选框**：位于行首，点击切换完成状态。
- **标题**：主文本区，单行超出省略。
- **元信息**：截止日期（如有）、优先级标签（高/中/低，颜色区分）。
- **备注**：展开后显示备注内容。

---

## 三、完成状态交互

### 3.1 完成动画（两步）

**第一步：视觉标记（300ms）**
- 待办项文字区域划上一道**斜线**。
- 文字颜色逐渐**置灰**（如 `Colors.grey`）。
- 复选框变为已勾选状态。

**第二步：位置移动（400ms）**
- 动画结束后，整条待办项以**渐隐+高度收缩**动画从当前位置移除。
- 同时在列表末尾以**渐显+高度展开**动画重新插入。
- 最终所有已完成待办项汇聚在列表底部。

### 3.2 取消完成
- 点击已完成的复选框，立即取消斜线和置灰，恢复原始样式。
- 待办项位置不变，仅在取消完成时若排序条件变化，下次列表刷新时重新排序。

## 四、排序规则

列表默认按以下**多级优先级**排序，权重从高到低：

| 优先级 | 排序维度 | 说明 |
|--------|----------|------|
| 1 | **置顶标记** | `isPinned == true` 的待办项排在最前面 |
| 2 | **今日截止** | 截止日期为今天的待办项排在置顶项之后 |
| 3 | **截止日期** | 有截止日期的按从近到远排序 |
| 4 | **无截止日期** | 无截止日期的排在最后 |
| 5 | **优先级** | 相同截止日期内按优先级排序：高 > 中 > 低 |
| 6 | **编辑时间** | 优先级相同则按 `updated_at` 降序，最近编辑的在前面 |

### SQL 排序示意

```sql
ORDER BY
  is_pinned DESC,                              -- 置顶优先
  CASE WHEN due_date = TODAY THEN 0 ELSE 1 END,-- 今日截止优先
  CASE WHEN due_date IS NULL THEN 1 ELSE 0 END,-- 有截止日期优先于无截止日期
  due_date ASC,                                 -- 截止日期从近到远
  CASE priority WHEN 'high' THEN 0 WHEN 'medium' THEN 1 ELSE 2 END,
  updated_at DESC                               -- 最后编辑时间降序
```

## 五、搜索与筛选

### 5.1 搜索框 + 类型选择框

```
┌────────────────────────────────────────┐
│ 🔍 输入关键词...      [全部类型 ▼]    │
└────────────────────────────────────────┘
```

- **搜索框**：支持按关键词模糊搜索待办项标题和备注内容。
- **类型选择框**：位于搜索框右侧，下拉选择待办事项类型（全部/工作/生活/学习/自定义1/自定义2）。
- **交集逻辑**：关键词搜索结果 与 类型筛选结果取**交集**，即同时满足两个条件才展示。
- 两个条件都为空/默认时，展示全部待办项。

## 六、待办创建弹窗

### 6.1 弹窗整体布局

**初始状态（未输入内容时）**

```
┌────────────────────────────┐
│  新建待办             [×]  │
├────────────────────────────┤
│                            │
│                            │
│ [☐] [输入待办事项...] [工作▼]│  ← 第一行
│                            │
│                            │
├────────────────────────────┤
│  ⏰ 设置日期       [完成]  │
└────────────────────────────┘
```

**输入内容回车后（清单模式）**

```
┌────────────────────────────┐
│  新建待办             [×]  │
├────────────────────────────┤
│  ── 待办清单 ──            │  ← 回车后出现的标题区
│  ☑ 完成季度报告            │
│  ☐ 预定会议室              │
│  ☐ 准备演示文稿            │
├────────────────────────────┤
│ [☐] [输入待办事项...] [工作▼]│  ← 第一行继续添加
│                            │
├────────────────────────────┤
│  ⏰ 设置日期       [完成]  │
└────────────────────────────┘
```

### 6.2 第一行控件（从左到右）

| 控件 | 说明 |
|------|------|
| **复选框** | 纯装饰作用，不可点击，视觉暗示待办项的完成态样式 |
| **输入框** | 隐藏边框，Hint 文字为 **"回车可连续添加待办事项"**。按回车键将当前内容加入上方"待办清单"列表，输入框清空，可继续输入下一条 |
| **类型选择框** | 下拉选择该待办项的类型，默认选中上次使用的类型 |

### 6.3 待办清单区域（回车后展开）

- 用户在第一行输入内容并按**回车**后，输入框上方展开一个标题为 **"待办清单"** 的区域。
- 回车的内容以清单条目形式出现在该区域，带有复选框样式（可勾选，但弹窗内不触发完成动画）。
- 可连续回车添加多条，每条独立展示。
- 清单区域内的条目可左滑删除，或长按编辑。

### 6.4 底部按钮栏

| 按钮 | 位置 | 说明 |
|------|------|------|
| **设置日期** | 左下角 | 闹钟图标 + "设置日期"文字，点击打开日期时间选择器，设置整个清单的截止日期 |
| **完成** | 右下角 | 主操作按钮，点击后将清单内所有条目批量写入数据库并关闭弹窗 |

**日期设置逻辑**
- 点击"设置日期"后弹出日期时间选择器（精确到年月日时分）。
- 设置成功后，按钮文字变为已选日期（如"12月25日 14:00"），闹钟图标高亮。
- 不设置日期时，清单无截止日期，按"无截止日期"规则排序。

**优先级设置**
- 弹窗内提供优先级选择（高/中/低），默认"中"。
- 所有在弹窗内连续添加的待办项共享同一优先级和截止日期。

### 6.5 类型选择框逻辑

#### 类型容量规则
- 默认内置 3 个类型：**工作**、**生活**、**学习**。
- 最多可再添加 **2 个自定义类型**，总计最多 **5 个类型**。

#### 下拉选项动态变化

**场景 A：仅有 3 个默认类型时**

```
工作
生活
学习
──────────
添加自定义类型
```

- 最后一项为 **"添加自定义类型"**。
- 点击后打开**类型编辑弹窗**。

**场景 B：已有 4~5 个类型时**

```
工作
生活
学习
我的项目A
我的项目B
──────────
编辑自定义类型
```

- 最后一项变为 **"编辑自定义类型"**。
- 点击后打开**类型编辑弹窗**，可修改自定义类型的名称。

### 6.6 类型编辑弹窗

```
┌────────────────────────────┐
│  编辑类型            [×]   │
├────────────────────────────┤
│ 自定义类型 1： [________]  │
│ 自定义类型 2： [________]  │
├────────────────────────────┤
│        [保存]  [取消]      │
└────────────────────────────┘
```

- 最多显示 2 个自定义类型输入框。
- 输入框预填充现有自定义类型名称（如有）。
- 保存时校验：名称非空、不重复、不超过 4 个字。
- 自定义类型被删除时，已关联的待办项类型回退为"未分类"或保留原值。

### 6.7 连续添加交互

1. 用户在输入框输入内容，按**回车键**。
2. 若这是第一条，输入框上方展开"待办清单"标题区。
3. 当前内容以临时条目形式展示在清单区域内。
4. 输入框清空，聚焦状态保持，Hint 文字不变。
5. 用户可继续输入下一条，重复上述过程。
6. 点击 **"完成"** 后，所有清单条目批量写入数据库并关闭弹窗。
7. 若清单为空（未添加任何条目），点击"完成"不创建任何数据，直接关闭弹窗。

## 七、数据表设计

### 7.1 待办项表（todos）

```sql
CREATE TABLE todos (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,           -- 待办事项标题
    content TEXT,                  -- 待办事项备注/详情
    type TEXT,                     -- 类型：工作/生活/学习/自定义
    due_date INTEGER,              -- 截止日期时间戳（毫秒），可为 NULL
    priority TEXT DEFAULT 'medium',-- 优先级：high / medium / low
    is_completed INTEGER DEFAULT 0,-- 0: 未完成, 1: 已完成
    is_pinned INTEGER DEFAULT 0,   -- 0: 未置顶, 1: 置顶
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);
```

### 7.2 类型配置表（todo_types）

```sql
CREATE TABLE todo_types (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,     -- 类型名称
    is_builtin INTEGER DEFAULT 0,  -- 1: 内置（工作/生活/学习）, 0: 自定义
    sort_order INTEGER             -- 展示顺序
);

-- 初始化数据
INSERT INTO todo_types (id, name, is_builtin, sort_order) VALUES
('work', '工作', 1, 0),
('life', '生活', 1, 1),
('study', '学习', 1, 2);
```

## 八、技术选型

| 需求 | 选型 | 说明 |
|------|------|------|
| 列表组件 | Flutter `ListView.builder` | 线性列表标配，支持大量数据懒加载 |
| 完成动画 | `AnimatedContainer` + `CustomPaint`（斜线） | 斜线用 CustomPaint 绘制，移动用 AnimatedContainer 控制高度/透明度 |
| 复选框 | `Checkbox` / `CheckboxListTile` | Flutter 内置组件，满足基础交互 |
| 状态管理 | `Riverpod` / `Provider` | 待办项增删改查、完成状态、排序、筛选状态同步 |
| 数据库 | `sqflite` | 沿用项目统一存储方案，支持复杂查询和排序 |
| 搜索高亮 | `flutter_highlighting` 或自研 `TextSpan` | 关键词匹配处用 `TextSpan` 高亮展示 |
| 弹窗 | `showDialog` + `AlertDialog` | 创建弹窗和类型编辑弹窗均用系统弹窗 |
