# 打卡模块需求与技术方案

## 一、模块定位

打卡模块为独立模块，聚焦**习惯养成**，核心流程为：创建打卡项 → 每日打卡（倒计时/正向计时/直接完成）→ 累计统计。与待办模块完全解耦。

---

## 二、打卡列表页

### 2.1 布局
- 采用**线性列表（ListView）**，每个打卡项独占一行。
- 一行内**左侧展示打卡内容**，**右侧是操作按钮**。
- 页面右下角固定**悬浮加号按钮（FAB）**，点击打开创建弹窗。

### 2.2 单条打卡项结构

**类型一：倒计时**

```
┌─────────────────────────────────────┐
│  📖 每日读书              [  开始  ] │
│  ⏱️ 倒计时 00:30:00                 │
│  累计: 12小时30分                   │
└─────────────────────────────────────┘
```

**类型二：正向计时**

```
┌─────────────────────────────────────┐
│  🏃 晨跑                [  开始  ] │
│  ⏱️ 正向计时                        │
│  累计: 5小时20分                    │
└─────────────────────────────────────┘
```

**类型三：不计时**

```
┌─────────────────────────────────────┐
│  💊 每日吃药            [  完成  ] │
│  ⏱️ 不计时                          │
│  今日已完成 ✅                       │
└─────────────────────────────────────┘
```

- **左侧内容区**：
  - 图标 + 打卡项名称（单行，超出省略）。
  - 第二行显示打卡类型标签：**⏱️ 倒计时 XX:XX:XX**、**⏱️ 正向计时**、**⏱️ 不计时**。
  - 第三行显示累计时长或今日完成状态。
- **右侧按钮区**：
  - **倒计时/正向计时**：显示"开始"按钮，点击跳转计时页面。
  - **不计时**：显示"完成"按钮，点击后直接标记今日完成（无需跳转计时页）。
  - 若该打卡项今日已完成，按钮变为"已完成"（置灰不可点）。

### 2.3 排序规则

| 优先级 | 排序维度 | 说明 |
|--------|----------|------|
| 1 | **置顶标记** | `isPinned == true` 的打卡项排在最前面 |
| 2 | **今日未完成** | 今日未打卡的排在已打卡的前面 |
| 3 | **累计时长** | 按 `total_duration` 降序 |
| 4 | **最近打卡** | 累计相同时按 `last_checkin_at` 降序 |

## 三、打卡计时页面

### 3.1 正向计时模式

```
┌────────────────────────────┐
│  [←]  晨跑          [⋮]   │
├────────────────────────────┤
│  🏷️ 健康 · 正向计时        │
│                            │
│         00:23:45           │  ← 本次计时（大字号）
│      ─────────────         │
│                            │
│   今日累计   历史累计      │
│   00:23:45   05:20:10      │
│                            │
│  [  结束打卡  ]            │
└────────────────────────────┘
```

- 从 0 开始正向累计，每秒递增。
- 点击"结束打卡"结算本次时长，保存记录。

### 3.2 倒计时模式

```
┌────────────────────────────┐
│  [←]  每日读书      [⋮]   │
├────────────────────────────┤
│  🏷️ 学习 · 倒计时          │
│                            │
│         00:12:30           │  ← 剩余时间（大字号）
│      ━━━━━━━━━━━━          │  ← 进度条
│      计划: 00:30:00        │
│                            │
│  [  结束打卡  ]            │
│  [+5分] [+10分] [+30分]   │  ← 快捷加时
└────────────────────────────┘
```

- 从预设时长开始倒计时，每秒递减。
- 倒计时到 0 时触发提醒（铃声 + 弹窗）。
- 提供快捷加时按钮（+5分/+10分/+30分）。
- 超时后显示"已超时 XX:XX"，可结束或继续加时。

### 3.3 计时机制（通用）

- 采用**记录时间戳 + 进入页面时计算**策略。
- 正向计时：记录 `start_time`，显示时长 = 当前时间 - `start_time`。
- 倒计时：记录 `start_time` 和 `end_time = start_time + plan_duration`，显示剩余 = `end_time` - 当前时间。
- 切后台不停止，重新进入时自动追平。

## 四、打卡创建弹窗

点击打卡列表页右下角**悬浮加号按钮（FAB）**打开创建弹窗。

### 4.1 弹窗布局（三种状态）

**状态一：倒计时**

```
┌────────────────────────────┐
│  新建打卡             [×]  │
├────────────────────────────┤
│  [输入打卡内容...        ] │  ← 第一行：输入框
│                            │
│  标签: [健康 ▼]            │  ← 标签行
│                            │
│  [● 倒计时] [ 正向计时]    │  ← 类型选择
│  [  不计时  ]              │
│                            │
│  时长: [10分] [30分] [自定义]│  ← 时间行
│        自定义: [___] 分钟   │
│                            │
│  [取消]          [创建]    │
└────────────────────────────┘
```

**状态二：正向计时**

```
┌────────────────────────────┐
│  新建打卡             [×]  │
├────────────────────────────┤
│  [输入打卡内容...        ] │  ← 第一行：输入框
│                            │
│  标签: [健康 ▼]            │  ← 标签行
│                            │
│  [  倒计时 ] [● 正向计时]  │  ← 类型选择
│  [  不计时  ]              │
│                            │
│  💡 适合碎片时间记录       │  ← 提示文字
│                            │
│  [取消]          [创建]    │
└────────────────────────────┘
```

**状态三：不计时**

```
┌────────────────────────────┐
│  新建打卡             [×]  │
├────────────────────────────┤
│  [输入打卡内容...        ] │  ← 第一行：输入框
│                            │
│  标签: [健康 ▼]            │  ← 标签行
│                            │
│  [  倒计时 ] [ 正向计时]   │  ← 类型选择
│  [● 不计时  ]              │
│                            │
│  💡 适合不需要计时的打卡   │  ← 提示文字
│     内容，比如吃药         │
│                            │
│  [取消]          [创建]    │
└────────────────────────────┘
```

### 4.2 控件说明

| 区域 | 说明 |
|------|------|
| **输入框** | 第一行，Hint 文字为 **"输入打卡内容"**。打卡项名称，最多 10 字。 |
| **标签行** | 输入框下方，"标签:" + 下拉选择框，可选健康/学习/习惯，可自定义。默认"学习"。 |
| **类型选择** | 标签行下方，三个选项：倒计时 / 正向计时 / 不计时，单选 pill 样式。默认选中"正向计时"。 |
| **时间行** | 仅"倒计时"类型时显示。提供三个快捷按钮：**10分钟**、**30分钟**、**自定义**。选中"自定义"时展开数字输入框，可输入任意分钟数。 |
| **提示文字** | "正向计时"时显示 **"💡 适合碎片时间记录"**；"不计时"时显示 **"💡 适合不需要计时的打卡内容，比如吃药"**。 |
| **底部按钮** | 左下角"取消"，右下角"创建"。 |

### 4.3 字段校验

| 字段 | 必填 | 规则 |
|------|------|------|
| **打卡内容** | 是 | 非空，1~10 字。 |
| **标签** | 否 | 从已有标签选择或新建。 |
| **类型** | 是 | 三选一。 |
| **时长** | 条件必填 | 选"倒计时"时必须设置时长；快捷选择 10/30 分钟或自定义输入。 |

## 五、数据表设计

### 5.1 打卡项表（habits）

```sql
CREATE TABLE habits (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,            -- 打卡项名称
    tag TEXT,                      -- 打卡标签
    habit_type TEXT NOT NULL,      -- 打卡类型：countdown / forward / none
    plan_duration INTEGER,         -- 计划时长（秒），倒计时类型专用
    total_duration INTEGER DEFAULT 0, -- 累计总时长（秒）
    today_duration INTEGER DEFAULT 0, -- 今日累计时长（秒）
    checkin_count INTEGER DEFAULT 0,  -- 总打卡次数
    today_count INTEGER DEFAULT 0,    -- 今日打卡次数
    is_pinned INTEGER DEFAULT 0,   -- 0: 未置顶, 1: 置顶
    is_timing INTEGER DEFAULT 0,   -- 0: 未在计时, 1: 正在计时
    current_record_id TEXT,        -- 当前进行中记录 ID
    last_checkin_at INTEGER,       -- 最后打卡时间戳
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);
```

### 5.2 打卡记录表（checkin_records）

```sql
CREATE TABLE checkin_records (
    id TEXT PRIMARY KEY,
    habit_id TEXT NOT NULL,        -- 关联打卡项 ID
    record_type TEXT NOT NULL,     -- countdown / forward / none
    start_time INTEGER NOT NULL,   -- 开始时间戳（毫秒）
    end_time INTEGER,              -- 结束时间戳（毫秒）
    duration INTEGER,              -- 本次时长（秒）
    is_completed INTEGER DEFAULT 0,-- 0: 未正常结束, 1: 正常结束
    created_at INTEGER NOT NULL,
    FOREIGN KEY (habit_id) REFERENCES habits(id) ON DELETE CASCADE
);
```

### 5.3 标签表（habit_tags）

```sql
CREATE TABLE habit_tags (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    sort_order INTEGER
);

-- 初始化

```

## 六、首页打卡统计卡片

### 6.1 卡片结构

```
┌────────────────────────────┐
│  📋 打卡                   │
├────────────────────────────┤
│  今日已打卡: 3 项          │
│  今日未打卡: 2 项          │
│  ────────                  │
│  📖 每日读书    ✅ 25分钟  │
│  🏃 晨跑        ⬜ 未打卡  │
│  💊 每日吃药    ✅ 已完成  │
└────────────────────────────┘
```

- 展示今日已打卡数量和未打卡数量。
- 下方展示 3~4 条打卡项快速预览，已打卡的展示时长/状态，未打卡的展示 ⬜。
- 点击卡片进入打卡列表页。

### 6.2 每日重置逻辑

- 每日 0 点时，自动重置所有打卡项的 `today_duration`、`today_count` 和今日完成状态。
- 可通过应用启动时检测日期变化实现重置。

## 七、技术选型

| 需求 | 选型 | 说明 |
|------|------|------|
| 列表组件 | `ListView.builder` | 线性列表 |
| 计时显示 | `Timer.periodic`（1秒） | 仅前台刷新 UI |
| 后台计时 | 记录时间戳 | 切后台不跑 Timer，恢复时计算时间差 |
| 倒计时结束提醒 | `flutter_local_notifications` | 倒计时到 0 时触发本地通知 |
| 状态管理 | `Riverpod` / `Provider` | 打卡状态、计时状态同步 |
| 数据库 | `sqflite` | 打卡项 + 打卡记录 |
| 每日重置 | 应用启动检测日期变化 | 检测到新日期时重置今日数据 |

## 八、与功能文档对应关系

| 功能文档需求 | 本方案实现 |
|-------------|-----------|
| 习惯打卡录入（5分） | 创建弹窗：名称 + 标签 + 三选一类型 + 计划时长（条件） |
| 累计计时功能（5分） | 正向计时：从 0 开始累计；倒计时：预设时长倒计时 |
| 打卡状态（5分） | `checkin_records` 每次打卡独立记录，首页展示今日已/未打卡 |
| 打卡时长统计（4分） | 日/周/月维度统计图表 |
| 打卡提醒（4分） | 倒计时结束本地通知提醒；每日定时推送通知 |
| 打卡排序（4分） | 置顶 → 今日未完成 → 累计时长 → 最近打卡 |
| 首页打卡统计（5分） | 首页卡片：已打卡/未打卡数量 + 快速预览 |
| 首页打卡快捷区（5分） | 点击卡片进入打卡列表，列表内一键开始/完成 |
