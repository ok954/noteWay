# 打卡计时功能技术方案

## 一、核心原理

采用**"记录时间戳 + 进入页面时计算"**的策略，不依赖后台持续运行 Timer，降低电量消耗和系统杀后台风险。

| 模式 | 计算方式 |
|------|----------|
| **正向计时** | 显示时长 = 当前时间 - `start_time`（每秒递增） |
| **倒计时** | 显示剩余 = `end_time` - 当前时间（`end_time = start_time + plan_duration`，每秒递减） |
| **不计时** | 无计时页面，点击"完成"直接生成为期 0 秒的打卡记录 |

---

## 二、技术选型

| 需求 | 选型 | 说明 |
|------|------|------|
| 计时显示 | `Timer.periodic`（1秒间隔） | 仅在前台页面打开时运行，用于刷新 UI 显示 |
| 后台保活 | **不依赖后台保活** | 记录时间戳后退出页面/切后台，重新进入时根据时间戳计算 |
| 倒计时结束提醒 | `flutter_local_notifications` | 注册本地通知，倒计时到 0 时触发铃声+弹窗 |
| 计时状态持久化 | `sqflite` | 打卡记录、计时中状态写入数据库，防崩溃丢失 |
| 时间计算 | `DateTime` / `Duration` | Dart 内置，无需额外依赖 |

## 三、数据表设计

### 3.1 打卡记录表（checkin_records）

```sql
CREATE TABLE checkin_records (
    id TEXT PRIMARY KEY,
    habit_id TEXT NOT NULL,        -- 关联的打卡项 ID
    record_type TEXT NOT NULL,     -- 记录类型：forward（正向计时）/ countdown（倒计时）/ none（不计时）
    start_time INTEGER NOT NULL,   -- 开始时间戳（毫秒）
    end_time INTEGER,              -- 结束时间戳（毫秒），结束时写入
    duration INTEGER,              -- 本次实际时长（秒）
    is_completed INTEGER DEFAULT 0,-- 0: 进行中/未正常结束, 1: 正常结束
    created_at INTEGER NOT NULL,
    FOREIGN KEY (habit_id) REFERENCES habits(id) ON DELETE CASCADE
);
```

### 3.2 打卡项表扩展（habits）

在既有 `habits` 表基础上，增加计时相关字段：

```sql
ALTER TABLE habits ADD COLUMN total_duration INTEGER DEFAULT 0;  -- 累计总时长（秒）
ALTER TABLE habits ADD COLUMN today_duration INTEGER DEFAULT 0;  -- 今日累计时长（秒）
ALTER TABLE habits ADD COLUMN today_count INTEGER DEFAULT 0;     -- 今日打卡次数
ALTER TABLE habits ADD COLUMN is_timing INTEGER DEFAULT 0;       -- 0: 未在计时, 1: 正在计时
ALTER TABLE habits ADD COLUMN current_record_id TEXT;            -- 当前进行中打卡记录的 ID
ALTER TABLE habits ADD COLUMN last_checkin_at INTEGER;           -- 最后打卡时间戳
```

## 四、正向计时流程

### 4.1 交互流程

```
用户在打卡列表点击"开始"
   ↓
创建 checkin_records 记录，record_type = 'forward'，start_time = 当前时间
   ↓
更新 habits.is_timing = 1, habits.current_record_id = 记录ID
   ↓
跳转计时页面，展示本次正向计时（大字号）+ 今日累计 + 历史累计
   ↓
用户点击"结束打卡"
   ↓
计算本次时长 = 当前时间 - start_time
   ↓
更新 checkin_records.end_time + duration + is_completed = 1
   ↓
更新 habits.total_duration += 本次时长
   ↓
更新 habits.today_duration += 本次时长
   ↓
更新 habits.today_count += 1
   ↓
更新 habits.last_checkin_at = 当前时间
   ↓
更新 habits.is_timing = 0, habits.current_record_id = null
   ↓
返回列表
```

## 五、倒计时流程

### 5.1 交互流程

```
用户在打卡列表点击"开始"
   ↓
创建 checkin_records 记录：
  record_type = 'countdown'
  start_time = 当前时间
  end_time = 当前时间 + habits.plan_duration
   ↓
注册本地通知：在 end_time 触发提醒
   ↓
更新 habits.is_timing = 1, habits.current_record_id = 记录ID
   ↓
跳转计时页面，展示剩余倒计时（大字号）+ 进度条 + 计划时长
   ↓
倒计时结束 / 用户手动点击"结束打卡"
   ↓
计算本次时长 = 实际结束时间 - start_time（不超过 plan_duration）
   ↓
更新 checkin_records.end_time + duration + is_completed = 1
   ↓
更新 habits.total_duration += 本次时长
   ↓
更新 habits.today_duration += 本次时长
   ↓
更新 habits.today_count += 1
   ↓
更新 habits.last_checkin_at = 当前时间
   ↓
更新 habits.is_timing = 0, habits.current_record_id = null
   ↓
取消本地通知（若已触发则无需取消）
   ↓
返回列表
```

```

│         00:12:30           │  ← 剩余时间（大字号，每秒递减）
│      ═════════════         │  ← 进度条（剩余比例）

│                            │
│  [  结束打卡  ]            │

│  [+5分] [+10分] [+30分]   │  ← 加时按钮
```

**倒计时结束状态**
- 圆环/进度条走满，颜色变为绿色。
- 触发本地通知（铃声 + 弹窗）。
- 按钮变为"任务完成"（绿色）。

**超时状态**
- 倒计时到 0 后若用户未操作，继续显示负向计时（已超时时长）。
- 进度条颜色变为橙色/红色警示。
- 快捷加时按钮高亮。

## 六、不计时流程

### 6.1 交互流程

```
用户在打卡列表点击"完成"
   ↓
直接创建 checkin_records 记录：
  record_type = 'none'
  start_time = end_time = 当前时间
  duration = 0
  is_completed = 1
   ↓
更新 habits.today_count += 1
   ↓
更新 habits.last_checkin_at = 当前时间
   ↓
列表刷新，按钮变为"已完成"（置灰）
```

### 6.2 特点

- **无计时页面**，点击即完成。
- 今日完成后按钮置灰不可点，次日重置后可再次完成。
- 打卡记录 duration 为 0，不计入累计时长统计。

## 七、核心代码逻辑

### 7.1 时间计算（不依赖后台 Timer）

```dart
class CheckinTimerService {
  // 正向计时：获取本次当前时长（秒）
  int getForwardDuration(CheckinRecord record) {
    final now = DateTime.now().millisecondsSinceEpoch;
    return ((now - record.startTime) / 1000).floor();
  }

  // 倒计时：获取剩余时间（秒）
  int getCountdownRemaining(CheckinRecord record) {
    final now = DateTime.now().millisecondsSinceEpoch;
    final remaining = ((record.endTime! - now) / 1000).floor();
    return remaining > 0 ? remaining : 0;
  }

  // 倒计时：获取已超时时间（秒）
  int getCountdownOvertime(CheckinRecord record) {
    final now = DateTime.now().millisecondsSinceEpoch;
    final overtime = ((now - record.endTime!) / 1000).floor();
    return overtime > 0 ? overtime : 0;
  }
}
```

### 7.2 前台 UI 刷新

```dart
Timer? _uiRefreshTimer;

void startUiRefresh() {
  _uiRefreshTimer = Timer.periodic(Duration(seconds: 1), (_) {
    if (habit.habitType == 'countdown') {
      // 倒计时模式
      final remaining = timerService.getCountdownRemaining(record);
      final overtime = timerService.getCountdownOvertime(record);
      setState(() {
        _remaining = remaining;
        _overtime = overtime;
        _isFinished = remaining == 0;
      });
    } else {
      // 正向计时模式
      final sessionDuration = timerService.getForwardDuration(record);
      final todayTotal = habit.todayDuration + sessionDuration;
      setState(() {
        _sessionDuration = sessionDuration;
        _todayDuration = todayTotal;
      });
    }
  });
}
```

## 八、切后台/退出页面处理

- 用户切后台或返回列表时，**不停止计时**，计时在后台"虚拟继续"。
- 再次进入计时页面时，根据 `start_time` / `end_time` 重新计算，UI 立即同步。
- 应用被系统杀死：下次启动时检查 `habits.is_timing == 1` 的记录：
  - 正向计时：提示"上次打卡未结束，是否继续？"，继续则恢复，结束则按当前时间结算。
  - 倒计时：若 `end_time` 已过，直接标记为完成并补提醒；若未过，恢复倒计时。

## 九、边界处理

| 场景 | 处理策略 |
|------|----------|
| **应用被杀死** | 重启时检查 `is_timing == 1`，正向计时提示恢复/结算，倒计时判断是否已过期 |
| **设备关机/重启** | 本地通知可能丢失，下次启动时根据 `end_time` 判断倒计时是否已过期，过期则补提醒 |
| **系统时间被修改** | 以 `checkin_records.start_time` 和 `end_time` 为基准，相对计算 |
| **同一打卡项多次打卡** | 支持多条 `checkin_records`，每次独立记录；不计时类型每次点击都生成新记录 |
| **倒计时加时** | 点击快捷加时按钮，更新 `end_time` 并重新注册本地通知 |
| **网络切换** | 纯本地计算，无网络依赖；联网后按笔记编辑方案的空闲上传策略同步打卡记录 |
| **多设备同步** | 打卡记录随打卡数据云端同步；正在计时的状态仅保存在本地当前设备 |
| **每日重置** | 应用启动时检测日期变化，重置 `today_duration` 和 `today_count` |
