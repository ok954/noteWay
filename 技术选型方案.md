# 记途 — 技术选型方案

## 一、总体架构

采用 **Flutter 跨端客户端 + 云后端服务** 的架构。客户端负责所有本地功能和离线使用，云端服务仅用于数据备份、同步和用户账户，遵循"默认无需登录"原则。

| 层级 | 技术方向 | 说明 |
|------|----------|------|
| 客户端 | Flutter (Dart) | 一套代码覆盖 iOS、Android、Windows |
| 服务端 | Node.js / Go / Java 均可 | 用户认证、数据同步、文件存储 |
| 数据库（服务端） | PostgreSQL / MySQL + Redis | 用户数据、会话缓存 |
| 对象存储 | 阿里云 OSS / AWS S3 / MinIO | 图片、备份文件 |

## 二、客户端技术选型

### 2.1 基础框架

| 技术 | 选型 | 版本建议 | 说明 |
|------|------|----------|------|
| **开发语言** | Dart | SDK ≥ 3.0 | Flutter 官方语言 |
| **UI 框架** | Flutter | ≥ 3.16 | 跨平台 UI 框架，单代码库覆盖多端 |
| **平台支持** | iOS / Android / Windows | — | 应用功能文档要求的三端 |

### 2.2 状态管理

| 技术 | 选型 | 说明 |
|------|------|------|
| **首选** | Riverpod | 类型安全、编译时检查、支持异步状态，Flutter 官方推荐方向 |
| **备选** | Provider | 社区成熟、学习成本低，若团队更熟悉可用 |
| **本地持久化状态** | SharedPreferences / Hive | 轻量配置存储（主题、字体、开关状态等） |

### 2.3 本地数据库

| 技术 | 选型 | 说明 |
|------|------|------|
| **数据库引擎** | sqflite | SQLite 的 Flutter 封装，成熟稳定，支持复杂查询 |
| **ORM/辅助** | sqflite_common_ffi（桌面端） | Windows 桌面端使用 FFI 版本，移动端用原生 sqflite |
| **数据库迁移** | 自研 Migration 脚本 | 版本迭代时管理表结构变更 |

**数据库文件结构**：
- 统一存储在应用私有目录，文件名 `app_database.db`
- 图片等大文件不存数据库，存文件系统，数据库存相对路径

### 2.4 富文本编辑（笔记模块）

| 技术 | 选型 | 说明 |
|------|------|------|
| **编辑器** | flutter_quill | Flutter 富文本编辑事实标准，支持加粗/斜体/下划线/图片插入/换行/分段 |
| **数据格式** | Quill Delta (JSON) | 编辑器原生格式，序列化后存入数据库 `content` 字段 |
| **纯文本提取** | quill_delta_parser | 从 Delta 提取纯文本，用于搜索预览和标题自动生成 |

### 2.5 列表与布局组件

| 技术 | 选型 | 说明 |
|------|------|------|
| **线性列表** | Flutter 内置 `ListView.builder` | 打卡列表、待办列表、设置列表 |
| **瀑布流** | flutter_staggered_grid_view | 笔记列表双列/多列瀑布流布局 |
| **分组列表** | 自研或 flutter_sticky_header | 设置页分组标题吸顶效果 |
| **进度条/滑块** | Flutter 内置 `LinearProgressIndicator`、`Slider` | 倒计时进度、云端存储空间 |
| **图表** | fl_chart | 打卡时长统计（日/周/月维度） |

### 2.6 文件与存储

| 技术 | 选型 | 说明 |
|------|------|------|
| **应用目录** | path_provider | 获取 documents、cache、应用私有目录 |
| **图片存储** | 应用私有目录 + 数据库存路径 | 图片复制到私有目录防外部删除，数据库存相对路径 |
| **文件选择** | file_picker | 导入数据时选择本地文件 |
| **图片选择** | image_picker | 笔记编辑时从相册/相机选图 |
| **文件分享** | share_plus | 分享应用、导出文件分享 |

### 2.7 通知与提醒

| 技术 | 选型 | 说明 |
|------|------|------|
| **本地通知** | flutter_local_notifications | 待办截止提醒、打卡倒计时结束提醒、每日打卡提醒 |
| **后台任务** | workmanager | 每日 0 点检测重置打卡数据（应用未启动时） |
| **音效播放** | audioplayers | 提醒音效自定义、打卡倒计时结束铃声 |

### 2.8 网络与云端

| 技术 | 选型 | 说明 |
|------|------|------|
| **HTTP 客户端** | dio | 功能完善，支持拦截器、并发、断点续传、文件上传下载 |
| **JSON 序列化** | json_serializable + build_runner | 自动生成 Model 的 toJson/fromJson |
| **网络状态** | connectivity_plus | 检测 Wi-Fi / 移动网络，控制云端同步策略 |

### 2.9 第三方登录

| 技术 | 选型 | 说明 |
|------|------|------|
| **微信登录** | fluwx | 微信 SDK Flutter 插件，支持登录、分享 |
| **QQ 登录** | tencent_kit | QQ SDK Flutter 插件 |
| **手机号登录** | 自研短信验证码 | 对接阿里云/腾讯云短信服务，服务端发码、客户端校验 |

### 2.10 数据导入导出

| 技术 | 选型 | 说明 |
|------|------|------|
| **CSV 生成** | csv | Dart CSV 编码解码库 |
| **JSON 处理** | dart:convert (内置) | 无需额外依赖 |
| **TXT 导出** | 自研格式化 | 按模块分组生成人类可读文本 |

### 2.11 其他工具库

| 技术 | 选型 | 说明 |
|------|------|------|
| **UUID 生成** | uuid | 生成笔记、待办、打卡、记录的唯一 ID |
| **日期时间** | intl | 国际化日期格式化、时间戳转本地时间 |

| **BI/埋点（可选）** | firebase_analytics 或自研 | 用户行为统计、崩溃分析 |

## 三、服务端技术选型

### 3.1 基础框架

| 技术 | 选型 | 说明 |
|------|------|------|
| **推荐方案 A** | Node.js + NestJS | TypeScript 全栈，开发效率高，社区生态丰富 |
| **推荐方案 B** | Go + Gin | 性能高、编译快、适合高并发，适合后期扩展 |
| **推荐方案 C** | Java + Spring Boot | 团队若熟悉 Java 生态，稳定可靠，企业级方案 |

以下以 **Node.js + NestJS** 方案为主进行描述。

### 3.2 数据库

| 技术 | 选型 | 说明 |
|------|------|------|
| **主数据库** | PostgreSQL | 关系型数据库，支持 JSONB 字段（灵活存储 Delta 内容），稳定可靠 |
| **ORM** | Prisma | 类型安全、自动迁移、支持 PostgreSQL/MySQL/SQLite |
| **缓存** | Redis | 会话缓存、短信验证码过期、限流、热点数据缓存 |

### 3.3 对象存储

| 技术 | 选型 | 说明 |
|------|------|------|
| **国内** | 阿里云 OSS / 腾讯云 COS | 国内用户访问速度快，SDK 完善 |
| **海外** | AWS S3 / Cloudflare R2 | 海外用户或全球化部署 |
| **私有部署** | MinIO | 自建对象存储，兼容 S3 协议 |

### 3.4 认证与安全

| 技术 | 选型 | 说明 |
|------|------|------|
| **Token 认证** | JWT (JSON Web Token) | Access Token + Refresh Token 双令牌机制 |
| **密码加密** | bcrypt / argon2 | 用户密码哈希存储 |
| **接口限流** | rate-limiter-flexible (Redis) | 防止短信轰炸、暴力破解 |
| **HTTPS** | Let's Encrypt / 云厂商证书 | 全站 HTTPS，强制 TLS 1.2+ |

### 3.5 短信服务

| 技术 | 选型 | 说明 |
|------|------|------|
| **国内** | 阿里云短信 / 腾讯云短信 | 稳定、价格低、接入文档完善 |
| **国际** | Twilio / Vonage | 海外手机号验证 |

### 3.6 部署与运维

| 技术 | 选型 | 说明 |
|------|------|------|
| **容器化** | Docker + Docker Compose | 开发环境统一，便于本地调试 |
| **编排（生产）** | Kubernetes / 云厂商容器服务 | 弹性伸缩、服务发现、负载均衡 |
| **CI/CD** | GitHub Actions / GitLab CI | 自动化构建、测试、部署 |
| **日志** | winston (Node) + ELK / 云日志 | 结构化日志收集与检索 |
| **监控** | Prometheus + Grafana | 服务指标监控与告警 |
| **APM** | Sentry / 阿里云 ARMS | 错误追踪、性能监控 |

## 四、数据同步协议

### 4.1 同步策略

| 策略 | 说明 |
|------|------|
| **增量同步** | 客户端记录每条数据的 `updated_at`，仅同步变更数据，减少流量 |
| **冲突解决** | 简单时间戳策略：本地 `updated_at` > 云端则覆盖，反之则拉取，相等则跳过 |
| **空闲上传** | 笔记编辑停止 30 秒后触发上传，快速连续编辑只上传最后一次 |
| **失败重试** | 上传失败标记为"待同步"，网络恢复或下次启动时批量补传 |

### 4.2 同步接口设计

```
POST /api/sync/upload
  请求: { notes: [...], todos: [...], habits: [...], lastSyncAt: timestamp }
  响应: { syncedIds: [...], conflicts: [...], serverTime: timestamp }

POST /api/sync/download
  请求: { lastSyncAt: timestamp }
  响应: { notes: [...], todos: [...], habits: [...], serverTime: timestamp }
```

## 七、选型理由总结

| 需求场景 | 为什么选这个方案 |
|----------|-----------------|
| 跨平台（iOS/Android/Windows） | Flutter 单代码库覆盖三端，开发效率高，UI 一致性最好 |
| 富文本编辑 | flutter_quill 是 Flutter 生态最成熟的富文本方案，Delta 格式便于序列化存储 |
| 本地离线优先 | sqflite + 应用私有目录，无需网络即可完整使用，符合"默认无需登录"原则 |
| 状态管理 | Riverpod 编译时安全、解耦彻底，适合中大型应用 |
| 云端同步 | REST API + JWT + 增量同步，实现简单，兼容性好 |
| 第三方登录 | 微信+QQ 覆盖国内主流用户，手机号作为兜底 |
| 文件导入导出 | CSV/TXT/JSON 三种格式兼顾通用性和完整性 |
