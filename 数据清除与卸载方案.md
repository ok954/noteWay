# 数据清除与卸载方案

## 一、Android 卸载时的默认行为

### 1.1 系统会自动清除的数据

Android 应用卸载时，系统**默认会自动删除**以下内容：

| 存储位置 | 是否自动清除 | 说明 |
|----------|-------------|------|
| `/data/data/<包名>/databases/` | ✅ 是 | SQLite 数据库 |
| `/data/data/<包名>/shared_prefs/` | ✅ 是 | SharedPreferences |
| `/data/data/<包名>/files/` | ✅ 是 | 应用私有文件（如笔记图片） |
| `/data/data/<包名>/cache/` | ✅ 是 | 缓存文件 |
| `getApplicationDocumentsDirectory()` | ✅ 是 | path_provider 获取的文档目录 |

**结论**：如果所有数据都存储在应用私有目录内，卸载应用即可自动清除所有数据，无需额外处理。

### 1.2 系统不会自动清除的数据

| 存储位置 | 是否自动清除 | 说明 |
|----------|-------------|------|
| 外部存储 `/sdcard/` | ❌ 否 | 如果应用把文件写到公共目录 |
| 相册/媒体库 | ❌ 否 | 插入到系统相册的图片 |
| 下载目录 | ❌ 否 | 导出的 CSV/TXT/JSON 文件 |
| Google 云端备份 | ❌ 否 | 如果开启了应用数据备份 |

## 二、确保卸载不留痕迹的配置

### 2.1 关闭 Android 数据备份

在 `android/app/src/main/AndroidManifest.xml` 中：

```xml
<application
    android:label="记途"
    android:name="${applicationName}"
    android:icon="@mipmap/ic_launcher"
    android:allowBackup="false"        <!-- 关闭云端备份 -->
    android:fullBackupOnly="false"
    tools:replace="android:allowBackup">
```

**说明**：
- `android:allowBackup="false"`：禁止 Google 云端备份应用数据
- 如果不设置，用户卸载后重装，系统可能自动恢复之前的数据

### 2.2 不要写入外部公共目录

确保图片和文件只存储在应用私有目录：

```dart
// ✅ 正确：存储在应用私有目录
directory = await getApplicationDocumentsDirectory();

// ❌ 避免：存储在公共外部存储
directory = await getExternalStorageDirectory();
```

### 2.3 导出文件的使用约定

用户导出数据时（CSV/TXT/JSON），文件会保存到用户选择的公共目录。这是用户主动行为，不属于应用残留数据，无需处理。

## 三、应用内"一键清除数据"功能

如果需要在不卸载应用的情况下清空所有数据，可以在设置页增加此功能。

### 3.1 UI 设计

在设置页"数据管理"分组中增加一项：

```
┌────────────────────────────┐
│  🗑️ 清除所有数据      >    │
│     将删除所有笔记、待办   │
│     和打卡数据（不可恢复） │
└────────────────────────────┘
```

点击后弹出二次确认弹窗：

```
┌────────────────────────────┐
│  ⚠️ 确认清除所有数据       │
├────────────────────────────┤
│                            │
│  此操作将永久删除：        │
│  · 所有笔记                │
│  · 所有待办事项            │
│  · 所有打卡项和记录        │
│  · 所有设置偏好            │
│                            │
│  此操作不可恢复！          │
│                            │
│  [取消]    [确认清除]      │
└────────────────────────────┘
```

### 3.2 核心实现代码

```dart
class DataClearService {
  final DatabaseHelper _db;
  final SharedPreferences _prefs;

  DataClearService(this._db, this._prefs);

  /// 清除所有应用数据
  Future<void> clearAllData() async {
    final db = await _db.database;

    // 1. 删除数据库所有表数据（保留表结构）
    await db.transaction((txn) async {
      await txn.delete('notes');
      await txn.delete('todos');
      await txn.delete('habits');
      await txn.delete('checkin_records');
      await txn.delete('todo_types');         // 保留内置类型，或一并重置
      await txn.delete('habit_tags');         // 保留内置标签，或一并重置
      await txn.delete('settings');
      await txn.delete('recycle_bin');
      await txn.delete('sync_records');
    });

    // 2. 重新插入初始化数据（内置类型、标签等）
    await _db.insertInitialData();

    // 3. 清除 SharedPreferences
    await _prefs.clear();

    // 4. 删除应用私有目录下的所有图片文件
    await _clearImageFiles();

    // 5. 取消所有已注册的本地通知
    await flutterLocalNotificationsPlugin.cancelAll();
  }

  /// 清除图片文件
  Future<void> _clearImageFiles() async {
    final appDir = await getApplicationDocumentsDirectory();
    final imagesDir = Directory('${appDir.path}/images');

    if (await imagesDir.exists()) {
      await imagesDir.delete(recursive: true);
      await imagesDir.create(); // 重建空目录
    }
  }
}
```

### 3.3 触发后处理

清除数据后需要：

```dart
Future<void> onConfirmClear(BuildContext context) async {
  showLoadingDialog(context, '正在清除...');

  try {
    await dataClearService.clearAllData();

    if (context.mounted) {
      Navigator.of(context).pop(); // 关闭 loading

      // 显示成功提示
      ScaffoldMessenger.of(context).showSnackBar(
        const SnackBar(content: Text('所有数据已清除')),
      );

      // 重置所有 Riverpod Provider 状态
      ref.invalidate(noteProvider);
      ref.invalidate(todoProvider);
      ref.invalidate(habitProvider);
      ref.invalidate(settingsProvider);

      // 可选：重启应用或跳转到首页
      Navigator.of(context).pushAndRemoveUntil(
        MaterialPageRoute(builder: (_) => const HomePage()),
        (route) => false,
      );
    }
  } catch (e) {
    if (context.mounted) {
      Navigator.of(context).pop();
      showErrorDialog(context, '清除失败: $e');
    }
  }
}
```

## 四、Windows 桌面端的数据清除

### 4.1 卸载时的默认行为

Windows 应用卸载时，数据是否被清除取决于安装包的配置：

| 安装方式 | 数据位置 | 卸载时是否清除 |
|----------|----------|---------------|
| MSIX 安装包 | `%LocalAppData%\Packages\<包名>` | ✅ MSIX 自动清除 |
| Inno Setup 安装包 | `%AppData%\<应用名>` | ❌ 需手动配置卸载脚本 |
| 绿色版（直接运行） | 应用同级目录 | ❌ 直接删除文件夹即可 |

### 4.2 推荐做法

**方案 A：使用 MSIX 打包（推荐）**

```bash
flutter pub add msix
```

在 `pubspec.yaml` 中配置：

```yaml
msix_config:
  display_name: 记途
  publisher_display_name: Your Name
  identity_name: com.yourcompany.memoapp
  msix_version: 1.0.0.0
  logo_path: assets/logo.png
  capabilities: internetClient
```

打包命令：
```bash
flutter pub run msix:create
```

MSIX 是 Windows 原生应用包格式，卸载时会**自动清除所有应用数据**。

**方案 B：Inno Setup 增加卸载清除脚本**

如果必须用 exe 安装包，在 Inno Setup 脚本中增加：

```pascal
[UninstallDelete]
Type: filesandordirs; Name: "{userappdata}\记途"
Type: filesandordirs; Name: "{localappdata}\记途"
```

## 五、总结

| 场景 | 是否需要额外处理 | 做法 |
|------|----------------|------|
| Android 卸载自动清除 | ❌ 不需要 | 确保 `allowBackup="false"`，不写入外部存储 |
| Android 应用内一键清除 | ✅ 需要开发 | 设置页增加"清除所有数据"按钮，调用清除服务 |
| Windows MSIX 卸载 | ❌ 不需要 | MSIX 自动处理 |
| Windows exe 卸载 | ✅ 需要配置 | Inno Setup 脚本增加 `[UninstallDelete]` |
| iOS 卸载 | ❌ 不需要 | iOS 卸载自动清除应用沙盒内所有数据 |

**核心原则**：

1. 所有数据（数据库、图片、配置）只存储在**应用私有目录**，不写入系统公共目录
2. Android 关闭 `allowBackup`，防止云端残留
3. 提供应用内"一键清除"功能，满足不卸载就想清空的需求
4. Windows 推荐使用 MSIX 格式，卸载最干净
