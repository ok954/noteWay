# 笔记列表页需求与技术方案

## 一、页面需求

### 1.1 布局方式
- 采用**瀑布流（Masonry）布局**，卡片高度根据内容自适应，多列排列。
- 列数适配：移动端默认 2 列，平板/桌面端根据屏幕宽度动态增加列数（如 3~4 列）。

### 1.2 排序规则
- **默认排序**：按 `updated_at` 时间戳降序，越晚编辑的笔记排在越前面。
- **置顶机制**：支持将单条笔记置顶，置顶笔记始终排在列表最前方；多条置顶笔记之间仍按 `updated_at` 降序排列。
- 排序优先级：`isPinned DESC` > `updated_at DESC`。

### 1.3 卡片展示规则

#### 整体结构（从上到下）
1. **图片缩略图区**（可选）：若笔记中包含图片，在卡片顶部展示首图缩略图；无图则不显示该区域。
2. **标题区**：卡片中部，单行展示。
3. **内容摘要区**：卡片底部，最多展示固定行数（根据瀑布流列数和卡片宽度自适应，建议 2~4 行）。

#### 各区域详细规则

| 区域 | 规则 |
|------|------|
| **图片缩略图** | 仅当笔记 `image_paths` 非空时展示；取第一张图片，等比缩放裁剪填满卡片宽度；图片高度固定（如 120dp），避免瀑布流高度差异过大。 |
| **标题** | 单行文本，超出部分用省略号（`TextOverflow.ellipsis`）。若用户未设置标题，则提取笔记正文纯文本的前 N 个字作为默认标题（N 建议 10 个字，与功能文档一致）。 |
| **内容摘要** | 展示笔记正文纯文本，行数根据瀑布流列数自适应（如 2 列时展示 3 行，3 列时展示 2 行）。超出部分用省略号。 |
| **短内容兜底** | 若笔记正文纯文本长度 ≤ 3 个字，则标题即为这 3 个字，标题下方的内容区展示文案 **"无更多文案"**。 |

#### 卡片样式
- 圆角卡片（建议 12dp），带轻微阴影，背景与主题色适配。
- 卡片间距：横向和纵向均保持固定间距（如 8dp），形成整齐的瀑布流视觉效果。

### 1.4 交互
- **点击卡片**：进入笔记编辑页。
- **长按卡片**：弹出操作菜单，包含「置顶/取消置顶」「删除」选项。
- **下拉刷新**：重新加载笔记列表。
- **空状态**：当没有任何笔记时，展示空状态插图和提示文案（如 "还没有笔记，点击右下角新建"）。

## 二、数据层调整

在既有 `notes` 表基础上，新增置顶字段：

```sql
ALTER TABLE notes ADD COLUMN is_pinned INTEGER DEFAULT 0;  -- 0: 未置顶, 1: 置顶
```

### 查询 SQL

```sql
SELECT * FROM notes
ORDER BY is_pinned DESC, updated_at DESC;
```

### 数据映射到卡片模型

```dart
class NoteCardModel {
  final String id;
  final String title;        // 用户标题或默认标题
  final String? thumbnail;   // 首图路径，无图则为 null
  final String summary;      // 内容摘要纯文本
  final bool isPinned;
  final DateTime updatedAt;
}
```

- `title` 生成逻辑：若 `notes.title` 非空则直接用；否则取 `notes.plain_text` 前 10 字。
- `summary` 生成逻辑：取 `notes.plain_text`，长度 ≤ 3 时卡片内容区显示 "无更多文案"；否则显示截取后的摘要。
- `thumbnail` 生成逻辑：若 `notes.image_paths` JSON 数组非空，取第一个路径。

## 三、技术选型

| 需求 | 选型 | 说明 |
|------|------|------|
| 瀑布流布局 | `flutter_staggered_grid_view` | Flutter 生态最成熟的瀑布流组件，提供 `MasonryGridView`，支持自适应高度、跨端一致，维护活跃。 |
| 图片加载与缓存 | `cached_network_image` / `flutter_cache_manager` | 本地图片同样可走缓存组件，避免列表滑动时重复解码导致卡顿；配合 `Image.file` 或统一封装。 |
| 状态管理 | `Riverpod` / `Provider` | 列表数据、置顶状态变化需实时同步到 UI。 |
| 数据库 | `sqflite` | 沿用笔记编辑方案，查询时按 `is_pinned DESC, updated_at DESC` 排序。 |
| 文本截断与省略 | Flutter 内置 `Text` Widget | `maxLines` + `overflow: TextOverflow.ellipsis` 即可满足单行/多行省略需求。 |

### 为什么不自己实现瀑布流

瀑布流需要精确计算每列累计高度、动态分配新卡片到最短列，并处理异步图片加载后的高度变化，自研工作量大且边界情况多（如快速滑动、图片加载前后高度跳动）。`flutter_staggered_grid_view` 已成熟解决这些问题。

## 四、核心代码结构示意

```dart
// 1. 列表页
MasonryGridView.count(
  crossAxisCount: crossAxisCount, // 根据屏幕宽度计算
  mainAxisSpacing: 8,
  crossAxisSpacing: 8,
  itemCount: notes.length,
  itemBuilder: (context, index) {
    return NoteCard(note: notes[index]);
  },
)

// 2. 卡片 Widget
class NoteCard extends StatelessWidget {
  final NoteCardModel note;

  @override
  Widget build(BuildContext context) {
    return Card(
      child: Column(
        crossAxisAlignment: CrossAxisAlignment.start,
        children: [
          if (note.thumbnail != null)
            ClipRRect(
              child: Image.file(File(note.thumbnail!), height: 120, fit: BoxFit.cover),
            ),
          Padding(
            padding: EdgeInsets.all(12),
            child: Column(
              crossAxisAlignment: CrossAxisAlignment.start,
              children: [
                Text(
                  note.title,
                  maxLines: 1,
                  overflow: TextOverflow.ellipsis,
                  style: titleStyle,
                ),
                SizedBox(height: 4),
                Text(
                  note.summary.length <= 3 ? '无更多文案' : note.summary,
                  maxLines: summaryMaxLines, // 根据列数动态设置
                  overflow: TextOverflow.ellipsis,
                  style: bodyStyle,
                ),
              ],
            ),
          ),
        ],
      ),
    );
  }
}
```

## 五、与功能文档对应关系

| 功能文档需求 | 本方案实现 |
|-------------|-----------|
| 笔记本地存储（5分） | sqflite 持久化，新增 `is_pinned` 字段 |
| 标题设置（4分） | 无标题时自动截取正文前 10 字作为卡片标题 |
| 编辑实时保存（4分） | 保存后 `updated_at` 更新，列表自动按时间重新排序 |
| 富文本编辑（4分） | 卡片中展示纯文本摘要，不渲染富文本样式，保持简洁 |
| 首页只保留待办事项统计 | 笔记独立为「笔记列表页」，通过底部导航或抽屉进入，不影响首页 |
